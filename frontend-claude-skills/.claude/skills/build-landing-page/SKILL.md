# Build Landing Page

Create **conversion-focused**, visually polished landing pages.

## Primary objective
Make the page feel like a premium product site that communicates value in under 5 seconds.

## Default stack
- Bun
- Next.js App Router
- TypeScript
- Tailwind CSS
- shadcn/ui when useful
- Spline only when a 3d hero meaningfully improves the page

## Build order
1. Write the offer clearly.
2. Define the hero headline, subheadline, and primary CTA.
3. Add proof: logos, testimonials, metrics, or use cases.
4. Add feature sections with strong scannability.
5. End with a compelling CTA and simple footer.

## Landing page rules
- The hero must instantly explain **what it is**, **who it is for**, and **why it matters**.
- Every section must support either clarity, trust, or conversion.
- Keep copy concise and high-signal.
- Make CTAs obvious and repeated naturally.
- Use mock screenshots or stylized UI blocks when product visuals are missing.

## Recommended section patterns
### SaaS
- hero
- trusted by / logos
- benefits
- product preview
- feature grid
- testimonials
- pricing
- faq
- final cta

### Personal brand
- hero
- authority proof
- featured work
- services or offers
- testimonials
- contact cta

### Waitlist / launch page
- hero
- core promise
- 3 benefit cards
- preview section
- email capture cta
- faq

## 3d landing page guidance
When the user asks for a 3d landing page:
- keep the 3d scene in the hero or product showcase area
- place the strongest headline and CTA opposite the scene
- use the Spline integration pattern from `build-stunning-ui`
- keep the rest of the page calmer so the 3d section remains the focal point
- preserve mobile readability by stacking content and reducing scene height

## Visual direction
- Use one strong accent color and a disciplined neutral palette.
- Favor large headings and generous spacing.
- Use gradients and spotlight effects with restraint.
- Make screenshots, cards, and 3d canvases feel tangible with depth.

## Avoid
- Generic lorem ipsum sections with no purpose
- Too many competing CTA styles
- Weak social proof placement
- Feature paragraphs that are too long
- Decorative 3d that harms readability or performance

## Deliverable
Return complete page files and briefly explain the conversion logic behind the layout.
